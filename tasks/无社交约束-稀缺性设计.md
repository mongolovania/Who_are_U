# 无社交属性的稀缺性设计：称号 & 外观系统

> 约束：App 无社交功能（无排行榜、无好友、无服务器用户数据）
> 设计原则：稀缺性不来自"比别人强"，而来自"自我的深度旅程"
> 日期：2026-06-07

---

## 一、核心设计哲学

```
传统社交 App 的稀缺感：  "我有这个称号，你没有" → 比较驱动
你谁啊的稀缺感：         "我走到了这一步，我遇见了这个自己" → 自我发现驱动
```

**四个稀缺性来源：**
1. **时间稀缺** — 只有在特定时刻才能触发（节日、深夜、坚持天数）
2. **条件稀缺** — 需要罕见的行为组合（低概率随机、高难度门槛）
3. **路径稀缺** — 基于画像特质的分叉（不同特质→不同称号树）
4. **深度稀缺** — 需要大量积累才能解锁（百次对话、年际回顾）

## 二、称号系统：三级 + 全域神话

### 2.1 称号分级体系

```
🏅 传说 (Legendary)    ← 全域唯一的神话级，极低概率或极苛刻条件
🏆 史诗 (Epic)         ← 区域隐藏，需要特定阶段+特质+运气
🎖️ 稀有 (Rare)         ← 条件触发，大部分人能获得但需要刻意追求
📛 普通 (Common)       ← 流程必然获得，基础成就
```

### 2.2 「世界级称号」的设计（无社交如何实现）

世界级称号 = **全域神话 (Legendary)**，通过以下机制实现稀缺：

#### 机制 A：概率掉落（幸运系统）

每次完成深度对话后，系统掷骰子：

| 掉落池 | 概率 | 内容 | 示例称号 |
|--------|------|------|---------|
| 普通池 | 15% | 随机普通称号 | "初入山林" |
| 稀有池 | 5% | 随机稀有称号 | "不眠的对话者" |
| 史诗池 | 1% | 随机史诗称号 | "天选之子·山巅相会" |
| 神话池 | 0.1% | 全球神话称号 | **"被山记住的人"** |

> 0.1% 意味着平均对话 1000 次才出一次——对重度用户来说，可能 1-2 年才遇到一次。这种稀缺感是真实的，不依赖社交。

#### 机制 B：深层条件触发

神话称号不靠运气，而靠极度罕见的条件组合：

```
称号：「千面旅人」
条件：完成 1000 次对话 AND 情绪标签覆盖全部 7 种以上
稀有度：全域神话
触发提示："山说，你已走过足够远的路。从此你不再是你，你是所有经过你的人。"

称号：「与山同醒」
条件：连续 365 天每天至少一次对话
稀有度：全域神话
触发提示："一年了。山记得你每一个深夜，每一个清晨。你已是山的一部分。"

称号：「独影之巅」
条件：我之山所有迷雾散去（全部区域解锁） AND 完成 50 次困难节点
稀有度：全域神话
```

#### 机制 C：时间之门

某些称号只在特定**时间段**开放，过了就等明年：

```
称号：「冬日不眠者」
开放期：每年冬至前后 3 天
条件：在此期间完成一次深度对话
后缀：每年随机变化 —— "冬日不眠者·2026 寒夜版"

称号：「春分·新生」
开放期：每年春分
条件：在此期间开启一个新的副峰目标
```

#### 机制 D：画像分叉

同一成就，不同画像特质的人获得**不同称号**，每个人只能看到属于自己的那一条分支：

```
成就：完成 100 次对话

点子王特质 → 称号：「灵感喷涌者」
坚韧者特质 → 称号：「沉默的跋涉者」
温柔者特质 → 称号：「百次倾听」
迷茫者特质 → 称号：「在雾中找到路的」
```

> 用户不知道其他分支存在——他们只觉得自己的称号很独特。这制造了"个人化的稀缺"。

### 2.3 「全区域隐藏称号」的设计

将「全域」重新定义为**山之区域**而非地理位置：

```
我之山的区域划分：

🌫️  迷雾区 (0-30 节点)    —— "雾中称号"系列
☀️  暖光区 (31-70 节点)    —— "行者称号"系列
🌌  星空区 (71-150 节点)   —— "星辰称号"系列
⚡  雷暴区 (困难节点密集)   —— "勇者称号"系列
🏔️  山巅区 (150+ 节点)     —— "传说称号"系列
```

每个区域有 3-5 个**隐藏称号**，只有进入该区域并满足特定条件才触发：

```
暖光区隐藏称号：「午夜的太阳」
条件：在暖光区阶段，于凌晨 2-4 点完成一次对话
含义：即使最暗的时刻，你也在向上走

星空区隐藏称号：「与群星的契约」
条件：在星空区阶段，连续 7 天每天对话
含义：你已经走了这么远，星空为你作证
```

## 三、隐藏外观系统

### 3.1 外观获取机制

| 机制 | 描述 | 稀有度 |
|------|------|--------|
| **成就解锁** | 获得特定称号后自动解锁对应外观 | 普通~稀有 |
| **时间限定** | 节日/季节限定外观，错过等一年 | 稀有~史诗 |
| **随机掉落** | 对话后低概率掉落「神秘布料」 | 稀有~史诗 |
| **画像专属** | 特定特质才能看到的外观（不同特质看到不同商店物品） | 史诗 |
| **深度解锁** | 到达山脉特定区域解锁 | 普通~史诗 |

### 3.2 「隐藏服饰」的具体设计

#### A. 节日限定（错过=绝版）

```
「霜雪斗篷」—— 冬至限定
开放期：每年冬至 ±3 天
获取方式：在开放期内完成一次深度对话
第二年冬至：复刻但颜色不同（2026 银白 / 2027 冰蓝）

「春樱冠」—— 春分限定
开放期：春分 ±3 天
获取方式：在此期间开启新副峰目标
```

#### B. 随机掉落（惊喜感）

```
对话后掉落系统：

普通掉落 (5%)：基础染色材料
稀有掉落 (2%)：「山灵的信物」→ 兑换稀有外观
史诗掉落 (0.5%)：「星空碎片」→ 兑换史诗外观
神话掉落 (0.05%)：「独影之泪」→ 兑换终极外观「透明斗篷·山之祝福」
```

用户永远不知道下一次对话会不会掉神话物品——这种**独自的期待感**不依赖社交。

#### C. 画像分叉（同一商店，不同商品）

```
商店中「头部」分类：

点子王 → 看到「灵感光环」「创意王冠」「灯泡帽」
坚韧者 → 看到「铁脊头盔」「沉默面具」「负重者冠」
温柔者 → 看到「倾听者羽饰」「温暖绒帽」「共情光环」
迷茫者 → 看到「寻路者斗笠」「星光引路灯」「迷雾面纱」

每种特质≈4件专属外观，3种特质≈12件，用户只能看到自己画像分支的4件
```

## 四、无社交的「全域感」实现总结

```
┌─────────────────────────────────────────────────────┐
│              稀缺性来源          实现方式              │
├─────────────────────────────────────────────────────┤
│  时间稀缺    │  节日限定    │  冬至/春分…限时触发     │
│  概率稀缺    │  随机掉落    │  0.05%~5% 概率池       │
│  路径稀缺    │  画像分叉    │  不同特质→不同称号树    │
│  深度稀缺    │  累积门槛    │  百次/千次/连续365天    │
│  区域稀缺    │  山之区域    │  迷雾区→暖光区→星空区   │
│  视觉稀缺    │  外观分叉    │  同一商店不同人看到不同  │
└─────────────────────────────────────────────────────┘
```

**关键心理洞察：** 用户不知道还有其他分支、还有其他掉落、还有其他称号。他们只觉得「我的山、我的旅程、我的故事」是独一无二的——这就够了。

实际上，所有条件都是真随机、真时间、真门槛，不是虚假的稀缺。一个有耐心的重度用户确实需要一年才能收集齐全域称号，这种**真实的稀缺感**比任何排行榜都更有力。

## 五、对 WBS 的影响

称号和外观系统不需要调整 WBS 结构，但实现策略需注意：

- **所有称号/外观逻辑纯本地** — 只依赖本地时间、本地数据库计数、本地画像特质
- **时间限定**依赖 `DateTime.now()`（本地时间），用户改系统时间可能作弊——但无社交属性，作弊只影响自己，可接受
- **概率掉落**使用 `dart:math` 随机数，种子可混合用户 ID + 日期保证一定确定性
- **画像分叉**在称号/外观查询时根据 PersonaTrait 动态过滤
