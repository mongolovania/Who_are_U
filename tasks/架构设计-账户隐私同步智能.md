## 一、账户系统设计

### 1.1 认证方式矩阵

| 方式 | 覆盖区域 | 隐私友好度 | 实现复杂度 | 推荐 |
|------|---------|-----------|-----------|------|
| **Email + 密码** | 全球 | ⭐⭐⭐⭐⭐ | 低 | ✅ 主力 |
| **Apple Sign In** | 全球(iOS必须) | ⭐⭐⭐⭐⭐ (Hide My Email) | 中 | ✅ 必需 |
| **Google Sign In** | 全球(除中国) | ⭐⭐⭐ | 中 | ✅ 可选 |
| **微信登录** | 中国 | ⭐⭐ (需合规) | 高 | ⚠️ 后期 |

### 1.2 账户模型

```python
# 最小化用户表——不知道用户是谁
@dataclass
class UserRecord:
    user_id: str          # 随机 UUID，不包含任何身份信息
    email_hash: str       # SHA-256(email + salt)，用于去重和登录
    public_key: str       # E2E 加密公钥 (Curve25519)
    created_at: datetime
    last_login_at: datetime
    
    # 以下字段全部加密存储，仅用户持有私钥可解密
    encrypted_email: bytes     # 用户邮箱（AES-256-GCM，用派生密钥加密）
    encrypted_recovery: bytes  # 恢复密钥（可选）
    
    # 数据驻留
    data_region: str      # "eu-west" | "us-east" | "ap-southeast" | "cn-north"
    
    # 同意管理
    consents: dict         # {"analytics": True, "aggregation": True, "model_training": False}
    
    # GDPR 权利
    deletion_requested_at: datetime | None
    data_export_requested_at: datetime | None
```

### 1.3 登录流程

```
用户输入 email
      │
      ▼
客户端: SHA-256(email + global_salt) → email_hash
      │
      ▼
POST /auth/login {email_hash, password_derived_key}
      │
      ▼
服务端: 验证 → 签发 JWT (user_id, exp, 无 email)
      │
      ▼
客户端: 用 password 派生 E2E 密钥对
      │ (如果首次: 生成新密钥对，公钥提交服务端)
      │
      ▼
客户端: 拉取密文数据 → 本地解密 → 恢复状态
```

### 1.4 密码派生密钥 (不存密码)

```
password + email + salt
    │
    ▼
Argon2id (memory=64MB, iterations=3, parallelism=4)
    │
    ├──→ auth_key (32 bytes) → 用于登录验证
    ├──→ encryption_key (32 bytes) → 用于 E2E 数据加解密
    └──→ recovery_key (32 bytes) → 备用恢复（可选展示给用户）
```

> 服务端只存 `auth_key` 的 bcrypt hash，不知道 password 和 encryption_key。

**绝对不会提取的数据**（即使用户同意）：

- 完整决策内容

## 三、端到端加密同步协议

### 3.1 密钥架构

```
用户密码
    │
    ▼ Argon2id
encryption_key (32 bytes)
    │
    ├──→ data_key = HKDF(encryption_key, "sync-data")
    ├──→ metadata_key = HKDF(encryption_key, "sync-meta")  
    └──→ search_key = HKDF(encryption_key, "sync-search")  # 可选：服务端密文搜索
```

### 3.2 同步数据格式

```json
{
  "sync_object": {
    "id": "uuid-v4",
    "type": "conversation_history | decision_report | mountain_node | persona_trait | ombre_bucket | title | appearance",
    "version": 1,
    "created_at": "2026-06-07T02:30:00Z",
    "updated_at": "2026-06-07T02:30:00Z",
    
    "encrypted_payload": {
      "ciphertext": "base64...",
      "nonce": "base64...",
      "algorithm": "AES-256-GCM"
    },
    
    "metadata": {
      "type": "conversation_history",
      "size_bytes": 12345,
      "content_hash": "sha256..."
    }
  },
  
  "signature": "Ed25519(user_id + payload_hash)"
}
```

### 3.3 冲突解决

```
本地数据变更:
  updated_at 递增
  本地 SQLite 记录 (id, updated_at, content_hash)

服务端:
  接收新版本 → 比较 updated_at
  ├── 客户端新 → 替换
  ├── 服务端新 → 返回服务端版本给客户端
  └── 同版本不同 hash → CRDT merge (仅对 ombre_brain 桶的 tags/importance)
```

### 3.4 数据删除

```
用户请求删除:
1. 客户端: 发送 DELETE 指令（用私钥签名）
2. 服务端: 验证签名 → 删除该用户所有密文数据
3. 服务端: 聚合数据中减去该用户的贡献（隐私预算回滚）
4. 服务端: 30 天内可恢复（GDPR 软删除期）
5. 30 天后: 物理删除，不可恢复
```

## 四、匿名聚合引擎：称号百分比 & 隐藏道具

### 4.1 差分隐私方案

```python
# 称号稀有度计算——不暴露任何个人数据

class DifferentialPrivacyAggregator:
    """
    使用差分隐私计算全局统计，保证个体不可识别。
    
    参数:
    - epsilon: 隐私预算 (越小越隐私，建议 0.1-1.0)
    - min_threshold: 最少用户数 (低于此数不报告)
    """
    
    def compute_title_rarity(
        self, 
        title_id: str, 
        epsilon: float = 0.5
    ) -> dict:
        """
        返回: {
            "title_id": "百难克星",
            "rarity_percent": 3.2,  # 已加噪声
            "rarity_tier": "史诗",   # 普通/稀有/史诗/神话
            "total_users_in_pool": "10000+"  # 模糊化
        }
        """
        # 1. 计算真实计数
        raw_count = self._count_users_with_title(title_id)
        total_users = self._count_total_users()
        
        # 2. 添加 Laplace 噪声 (差分隐私核心)
        noisy_count = raw_count + self._laplace_noise(1.0 / epsilon)
        
        # 3. 阈值检查 (不足 100 人不显示百分比)
        if raw_count < 100:
            return {"rarity_tier": "???", "message": "此称号过于稀有，暂无统计"}
        
        # 4. 模糊化输出
        percentage = (noisy_count / total_users) * 100
        percentage = max(0.1, percentage)  # 不低于 0.1%
        
        return {
            "title_id": title_id,
            "rarity_percent": round(percentage, 1),
            "rarity_tier": self._tier_from_percent(percentage),
            "total_users_in_pool": self._blur_count(total_users)
        }
    
    def _laplace_noise(self, scale: float) -> float:
        """拉普拉斯噪声"""
        import random
        u = random.random() - 0.5
        return -scale * (1 if u < 0 else -1) * __import__('math').log(1 - 2 * abs(u))
    
    def _tier_from_percent(self, pct: float) -> str:
        if pct < 1:   return "神话"   # < 1%
        if pct < 5:   return "史诗"   # 1-5%
        if pct < 15:  return "稀有"   # 5-15%
        return "普通"                  # > 15%
    
    def _blur_count(self, n: int) -> str:
        """模糊化总用户数"""
        if n < 1000: return "1000+"
        if n < 10000: return "5000+"
        if n < 100000: return "10000+"
        return "100000+"
```

### 4.2 隐藏道具发放机制

```
隐藏道具获取条件（服务端计算，客户端执行）：

1. 绝对稀有度触发
   "当某称号的持有率 < 1% 时，持有者自动解锁隐藏外观'凤毛麟角'"

2. 时间窗口 + 稀有度
   "冬至期间，称号稀有度排前 5% 的称号持有者可解锁限定外观"

3. 匿名比较触发
   "你的'我之山'节点数超过 90% 的用户 → 解锁隐藏称号'独行者之巅'"
   (通过差分隐私计算分位数，不暴露其他用户具体数值)

4. 随机惊喜
   "每次对话后 0.05% 概率掉落隐藏外观碎片"
   (概率由服务端聚合数据动态调整，保持全局稀缺性)

5. 情绪共振
   "如果本周你的情绪趋势 (valence上升) 匹配全局'正在好转'模式 → 解锁'曙光'限定外观"
```

### 4.3 隐私保护下的排行榜思路

不显示排名，显示「你在哪里」：

```
传统社交:  "你排第 1,234 名"  ← 需要知道所有人
你谁啊:    "你的节点数超过了 90% 的同行者"  ← 只需要分位数
你谁啊:    "持有此称号的人，平均花费了 8 个月"  ← 只需要聚合统计
你谁啊:    "本周只有 0.3% 的人在凌晨 3 点对话"  ← 差分隐私
```

### 5.2 对话流程 + Ombre Brain

```
用户发起对话
    │
    ▼
[对话前] breath(user_id)
    │  浮现未解决记忆、高权重记忆
    │  "上次你纠结要不要离职...现在有新进展吗？"
    │
    ▼
[对话中] 5 阶段引导对话
    │  每个阶段关键洞察 → hold()
    │  ├── 阶段1 共情: hold("用户表达了职业倦怠", arousal=0.7)
    │  ├── 阶段2 自扫: hold("回忆起大学时热爱写作", valence=0.6)
    │  ├── 阶段3 拆解: hold("在稳定和热爱之间犹豫", arousal=0.8)
    │  └── ...
    │
    ▼
[对话后] dream()
    │  消化本次对话记忆
    │  检测到 "离职" 议题出现 3 次 → 提示升级为钉选准则
    │  写 feel: "这用户一直在纠结同一件事，但每次都有新发现"
    │
    ▼
[下次对话前] breath()
    │  浮现之前的 feel: "记得你上次说..."
    │  浮现未解决的记忆: "你还在考虑这件事..."
    │  让用户感受到「你记得我」
```

### 5.3 Ombre Brain 的多设备同步

```
设备 A (iPhone)
    │  Ombre Brain buckets/
    │  ├── dynamic/职业/离职纠结.md  (E2E encrypted)
    │  └── feel/2026-06-07.md
    │
    ▼ Sync (密文)
Server (密文存储)
    │
    ▼ Sync (密文)
设备 B (iPad)
    │  解密 → 本地 buckets/
    │  对话中 breath() → 浮现相同记忆
```

### 5.4 Ombre Brain 的改进方向

基于用户提出的「改进 ombre_brain」：

| 改进点 | 当前 Ombre Brain | 你谁啊定制版 |
|--------|-----------------|------------|
| **多用户隔离** | 单用户 buckets 目录 | 每用户独立命名空间 + E2E 加密 |
| **对话感知** | 通用 MCP 工具 | 与 5 阶段对话引擎深度耦合 |
| **决策记忆** | 通用记忆桶 | 新增 `decision` 类型桶（决策专用） |
| **情绪追踪** | 单点情绪标记 | 对话前后情绪变化追踪 (delta) |
| **称号联动** | 无 | 记忆里程碑触发称号解锁 |
| **聚合分析** | 无 | 匿名情绪趋势分析（差分隐私） |

## 八、总结

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 认证方式 | Email + Apple Sign In | 全球覆盖 + iOS 合规要求 |
| 数据加密 | E2E (AES-256-GCM + Curve25519) | 服务端永远看不见明文 |
| 密钥派生 | Argon2id (password → auth+encryption) | 抗暴力破解，服务端无密码 |
| 聚合模型 | 差分隐私 (Laplace noise) | 保护个体，统计可用 |
| 数据驻留 | 4 区域 (EU/CN/US/SG) | 满足 GDPR/PIPL/CCPA/PDPA |
| 记忆引擎 | Ombre Brain 定制版 | 已有的成熟情感记忆模型 |
| 删除机制 | 软删除 30 天 → 物理删除 | GDPR 合规 |

### 与 Ombre Brain 的关系

Ombre Brain 从「Claude 的辅助记忆工具」升级为「你谁啊用户的情感记忆引擎」：
- 当前：Claude ← MCP → Ombre Brain（1:1，单用户）
- 升级后：你谁啊 App ← 对话引擎 ← Claude + Ombre Brain（N:1:1，每用户独立记忆空间）
- Ombre Brain 管理「时间中发生的事」，用户画像管理「你是谁」，两层配合形成完整用户理解
