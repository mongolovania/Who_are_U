## 二、交付策略：底座框架先行

> **底座定义（2026-06-09 纠正）**：底座 = 关键技术突破的代码框架优先搭建。Memory Palace 记忆引擎框架、决策智囊引擎（24原型+50大师+选择引擎）、LLM Gateway 路由/缓存框架——这些是项目的核心技术突破，构成底座。用户可见层（我之山·对话UI·订阅）在底座框架之上搭建；基础设施层（账户·E2E同步·合规）在 v2 交付。

```
MVP v1 (14w·底座框架+可见层)      MVP v2 (+9w·基础设施+完善)    Phase 2+
框架骨架·技术突破·用户可见        账户+同步+聚合+合规+完整版    增值扩展
─────────────────────────────────────────────────────────────────────→
Week 1──────────14              Week 15─────────────23           Week 24+
                                                                       
✅ Memory Palace v0.9.0 框架        ✅ 账户系统+E2E同步             称号/外观/记忆
✅ 决策智囊框架(24原型+50大师)   ✅ Memory Palace 完整版(Ombre)  独影动画/副峰
✅ LLM Gateway(路由+缓存+SSE)   ✅ 决策智囊完整版(5阶段+报告)    Android适配
✅ 我之山+基础对话+订阅         ✅ 匿名聚合引擎(称号%)          运营后台
✅ 基础称号+基础外观            ✅ 全球合规(4区域)
```

### 为什么底座框架先行？

| 理由 | 说明 |
|------|------|
| **核心技术验证** | Memory Palace 和决策智囊是项目最大的技术不确定性，尽早搭建框架验证可行性 |
| **框架即约束** | 先建好记忆引擎和决策引擎的 API/接口契约，对话引擎和我之山才有明确的集成点 |
| **渐进增强** | 底座框架 → v1 基础可用 → v2 完整版，每一步都有可工作的系统 |
| **基础设施独立** | 账户/E2E/同步是可独立验证的系统，不与核心技术突破耦合，可并行或在 v2 交付 |

## 三、功能分层（按订阅视角）


### 月付会员层（订阅解锁）

```
会员层：持续价值交付，支撑续费
├── 🧠 无限决策对话
│   ├── 5阶段引导对话引擎（不限制次数）
│   ├── 完整决策报告（含历史回顾）
│   ├── 对话历史永久保留（MVP v1 本地加密 → v2 E2E 同步）
│   ├── 情绪标签自动标记（EmotionTag）
│   └── Ombre Brain 记忆引擎（v2 集成：breath→hold→dream 循环）
│
├── 🏔️ 完整我之山
│   ├── 全山可视化（迷雾散去，完整山脉视野）
│   ├── 所有对话自动生成节点
│   ├── 困难节点纪念碑（独影动作定格）
│   ├── 副峰系统（自定义小目标追踪）
│   └── 年度山脉回顾（成长轨迹总览）
│
├── 💬 生动陪伴与持久记忆
│   ├── 对话情感化设计（温暖语气、适时沉默、个性化回应）
│   ├── 情绪曲线回顾（每周/月度情绪变化可视化）
│   ├── 独影状态变化（随使用深度逐渐清晰）
│   └── 时光轴视图（按时间排列所有节点与故事）
│
├── 🏅 隐藏称号系统
│   ├── 基础成就（首次决策、十次对话…解锁普通称号）
│   ├── 隐藏称号（特殊条件触发，如"深夜倾诉者""百难克星"…）
│   ├── 稀有称号（v2 聚合引擎驱动：时间窗口+绝对稀有度）
│   └── 称号展示（在独影旁显示当前佩戴称号）
│
└── 🎨 外观装扮
    ├── 独影外观商店（帽子、眼镜、斗篷、配饰…）
    ├── 限定外观（v2 差分隐私触发：节日限定、稀有度解锁）
    ├── 外观组合搭配（layer 叠加系统）
    └── 独影换装在山上实时预览
```
