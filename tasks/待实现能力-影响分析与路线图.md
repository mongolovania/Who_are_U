## 三、5 项待实现能力对 6 维 Benchmark 的严格影响分析

> 分析方法：对每项待实现能力，逐一论证其对 6 个认知维度（简单回忆、多跳推理、时间推理、情感记忆、因果推理、跨引用）的影响机制。判定标准：影响检索质量 = 影响跑分；不影响检索质量 = 不影响跑分。

### 3.1 差分隐私先验层（v0.6.0 L2）

**机制分析**：差分隐私先验是从聚合用户数据中提取的群体统计（加噪后），用作 COLD/WARM 阶段的默认参数。当前系统使用 LLM 知识先验（`global_prior.py`）完成同一功能。差分隐私先验是**先验来源的替换/增强**，不是检索机制的变更。

| 维度 | 影响评估 | 论证 |
|------|---------|------|
| 简单回忆 | **无影响** | 简单回忆依赖 bm25+vector 路径——不受先验来源影响。DP 先验可能微调默认检索参数（如 top_k），但 COLD/WARM 阶段 BM25 基线不变。 |
| 多跳推理 | **无影响** | 多跳推理依赖 BFS+PPR+Community 图遍历——不受先验来源影响。DP 先验可能影响 Working Self 的初始化（从"LLM 通用知识"变为"本 App 用户群统计"），但不改变图遍历机制。 |
| 时间推理 | **无影响** | 时间推理依赖双时序模型+时间约束解析+叙事时间线——不受先验来源影响。 |
| 情感记忆 | **无影响** | 情感记忆依赖 4 层情感体系——不受先验来源影响。DP 先验可能提供更本地化的情绪分布基线（"本 App 用户的日常 valence 分布在 0.4-0.7"），但 LLM 知识先验已提供合理基线。 |
| 因果推理 | **无影响** | 因果推理依赖 causal typed edges + v0.7.0 6 模块——不受先验来源影响。 |
| 跨引用 | **无影响** | 跨引用依赖 typed edges + 多对多图 + cross_ref 路径——不受先验来源影响。 |

**结论：零影响。** 差分隐私先验是 LLM 知识先验的本地化校准——改变的是先验统计的来源，不是检索架构的任何组件。激活后可能在 COLD/WARM 阶段提供更精准的默认参数，但不会改变当前 6 维跑分。

### 3.2 Prompt 缓存

**机制分析**：Prompt 缓存是 LLM 调用时的前缀缓存优化——将静态 Prompt 模板缓存，避免重复发送相同的系统指令。这是纯粹的**成本优化**（减少输入 token），不涉及任何检索逻辑变更。

| 维度 | 影响评估 | 论证 |
|------|---------|------|
| 简单回忆 | **零影响** | Prompt 缓存不改变检索流程、不改变检索路径、不改变检索结果。唯一变化是 LLM 看到的是"缓存命中的 Prompt"而非"新发送的 Prompt"——内容完全一致。 |
| 多跳推理 | **零影响** | 同上。 |
| 时间推理 | **零影响** | 同上。 |
| 情感记忆 | **零影响** | 同上。 |
| 因果推理 | **零影响** | 同上。 |
| 跨引用 | **零影响** | 同上。 |

**结论：零影响。** Prompt 缓存是纯粹的工程成本优化，与检索质量和 Benchmark 跑分无关。类比：更换服务器的电源不会改变其上运行的算法的输出。

| 简单回忆 | **零影响** | SSE 不改变 `breath()` 或 `search_memory()` 的任何内部逻辑。检索结果完全一致——只是分批发送而非一次性发送。 |
| 多跳推理 | **零影响** | 同上。 |
| 时间推理 | **零影响** | 同上。 |
| 情感记忆 | **零影响** | 同上。 |
| 因果推理 | **零影响** | 同上。 |
| 跨引用 | **零影响** | 同上。 |

**结论：零影响。** SSE 是传输层优化——改变的是用户体验（更快的首字节时间），不是检索质量。当前 Gateway 已支持 `chat_stream()`，仅需 API 层暴露端点。

### 3.4 对话压缩器

**机制分析**：对话压缩器将超长对话历史压缩为更短的摘要形式，防止 LLM 上下文窗口溢出。**这是唯一有潜在影响的待实现能力**——需要严格分析压缩是否可能丢失关键检索上下文。

#### 3.4.1 压缩器的操作范围

对话压缩器作用于**活跃对话上下文**（当前会话中 LLM 的上下文窗口），不作用于**已存储的记忆**。区分：
- **活跃上下文**：当前对话中尚未被 `hold()` 存储的消息——LLM 需要在上下文中看到的
- **已存储记忆**：已经通过 `hold()` 存入 BucketManager 的记忆——不可变的，不受压缩器影响

#### 3.4.2 分维度影响分析

| 维度 | 影响评估 | 论证 |
|------|---------|------|
| 简单回忆 | **极低风险** | 简单回忆主要依赖已存储记忆的 BM25+vector 检索。如果压缩器在 `breath()` 之前运行，压缩后的查询可能丢失细节——但 `breath()` 使用的是原始 query，不是压缩后的对话。**存储侧不受影响。** |
| 多跳推理 | **低风险** | 多跳推理需要 LLM 在上下文中进行多步推理。如果压缩器将中间推理步骤合并/丢失，可能打断推理链。缓解措施：保留推理步骤标记（"第1步发现...第2步推断..."），压缩时保留结构化推理链。 |
| 时间推理 | **极低风险** | 时间推理依赖已存储记忆中的时间戳和叙事时间线——压缩器不影响这些。如果压缩器合并了多个时间段的对话，可能模糊时间参照，但不会改变已存储记忆的时间元数据。 |
| 情感记忆 | **低风险** | 情感记忆依赖已存储记忆中的 valence/arousal 值——压缩器不影响这些。但如果压缩器将高情感强度的对话摘要为中性摘要，`hold()` 时可能丢失情感信号。缓解措施：压缩时保留情感标记（valence/arousal 极值不压缩）。 |
| 因果推理 | **低风险** | 因果推理依赖已存储的 causal typed edges——压缩器不影响这些。但如果活跃上下文中正在构建因果链（"如果A→B→C"），压缩可能丢失中间步骤 B。缓解措施：压缩时检测因果连接词并保留完整因果链。 |
| 跨引用 | **极低风险** | 跨引用依赖已存储记忆之间的图关系——压缩器不影响这些。 |

#### 3.4.3 风险缓解措施

```
1. 存储优先原则：hold() 在压缩前执行——原始对话先存入，再压缩
2. 情感保留：valence/arousal 极值（|v|>0.7 或 a>0.8）的对话片段不压缩
3. 因果链保留：检测到因果连接词（"因为""所以""导致""结果是"）的片段保留完整
4. 推理链保留：结构化推理步骤保留标签和结论，可压缩中间计算
5. 时间锚点保留：时间参照（"上周""去年夏天""在我跳槽之前"）不压缩
6. 验证：压缩后运行当前 Benchmark 套件，确保无退化
```

**结论：低风险·可控。** 对话压缩器是唯一有潜在影响的待实现能力，但影响范围局限于活跃上下文（非已存储记忆）。通过 6 项缓解措施可将风险降至极低。实施前必须通过当前 Benchmark 套件验证无退化。
