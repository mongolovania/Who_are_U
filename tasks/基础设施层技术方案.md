### 1.2 用户表设计（最小化·不知道用户是谁）

```python
# PostgreSQL 用户表
CREATE TABLE users (
    user_id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email_hash      VARCHAR(64) NOT NULL UNIQUE,    -- SHA-256(email + global_salt)
    auth_key_hash   VARCHAR(128) NOT NULL,           -- bcrypt(auth_key)
    public_key      TEXT NOT NULL,                   -- Curve25519 公钥 (Base64)
    data_region     VARCHAR(20) NOT NULL,            -- eu-west | cn-north | us-east | ap-southeast
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_login_at   TIMESTAMPTZ,
    deletion_requested_at TIMESTAMPTZ,               -- GDPR 软删除标记
    data_export_requested_at TIMESTAMPTZ,

    -- 加密存储（用 encryption_key 加密，服务端无法解密）
    encrypted_email     BYTEA,          -- AES-256-GCM(email, encryption_key)
    encrypted_recovery  BYTEA,          -- 恢复密钥（可选）

    -- JSON 列
    consents        JSONB NOT NULL DEFAULT '{
        "analytics": false,
        "aggregation": false,
        "model_training": false
    }'
);

-- 索引
CREATE INDEX idx_users_email_hash ON users(email_hash);
CREATE INDEX idx_users_data_region ON users(data_region);
CREATE INDEX idx_users_deletion ON users(deletion_requested_at)
    WHERE deletion_requested_at IS NOT NULL;
```

### 1.3 密码派生密钥（Argon2id）

```
用户输入: password + email + global_salt
    │
    ▼
Argon2id (memory=64MB, iterations=3, parallelism=4)
    │
    ├──→ auth_key (32 bytes)
    │       │ bcrypt(salt=12) → 服务端存储
    │       └── 用于登录验证
    │
    ├──→ encryption_key (32 bytes)
    │       │ 永不离开客户端
    │       └── 用于 E2E 数据加解密
    │           ├── data_key = HKDF(encryption_key, "sync-data")
    │           ├── metadata_key = HKDF(encryption_key, "sync-meta")
    │           └── search_key = HKDF(encryption_key, "sync-search")
    │
    └──→ recovery_key (32 bytes)
            │ 可选展示给用户（助记词形式）
            └── 用于账号恢复
```

**关键安全属性：**
- 服务端只存 `bcrypt(auth_key)`，不知道 password 和 encryption_key
- 即使用户表泄露，攻击者需先破解 bcrypt 再破解 Argon2id
- encryption_key 永不离开客户端——服务端无法解密用户数据

## 二、E2E 加密同步协议

### 2.1 密钥架构

```
用户密码
    │
    ▼ Argon2id
encryption_key (32 bytes) —— 永不离开客户端
    │
    ├──→ data_key = HKDF-SHA256(encryption_key, "sync-data-v1")
    │       用于加密同步数据负载
    │
    ├──→ metadata_key = HKDF-SHA256(encryption_key, "sync-meta-v1")
    │       用于加密同步元数据（类型·大小·hash）
    │
    └──→ search_key = HKDF-SHA256(encryption_key, "sync-search-v1")
            (留作未来服务端密文搜索)
```

### 2.2 同步数据格式 (SyncObject)

```json
{
  "sync_object": {
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "user_id_hash": "sha256...",
    "type": "conversation_history",
```

### 2.3 加密/解密实现

```dart
// 客户端
class SyncCrypto {
  final Uint8List dataKey;      // 32 bytes
  final Uint8List metadataKey;  // 32 bytes

  /// 加密同步对象
  SyncObject encrypt(String type, Map<String, dynamic> payload) {
    final payloadBytes = utf8.encode(jsonEncode(payload));
    final payloadNonce = _generateNonce();  // 12 bytes random
    final ciphertext = AesGcm.encrypt(
      plaintext: payloadBytes,
      key: dataKey,
      nonce: payloadNonce,
    );

    final metadataBytes = utf8.encode(jsonEncode({
      'type': type,
      'size_bytes': payloadBytes.length,
      'content_hash': sha256(payloadBytes),
    }));
    final metadataNonce = _generateNonce();
    final encryptedMeta = AesGcm.encrypt(
      plaintext: metadataBytes,
      key: metadataKey,
      nonce: metadataNonce,
    );

    return SyncObject(
      id: uuid.v4(),
      type: type,
      encryptedPayload: EncryptedBlob(ciphertext, payloadNonce),
      encryptedMetadata: EncryptedBlob(encryptedMeta, metadataNonce),
      signature: _sign(id, sha256(ciphertext)),
    );
  }

  /// 解密同步对象
  Map<String, dynamic> decrypt(SyncObject obj) {
    final plaintext = AesGcm.decrypt(
      ciphertext: obj.encryptedPayload.ciphertext,
      key: dataKey,
      nonce: obj.encryptedPayload.nonce,
    );
    return jsonDecode(utf8.decode(plaintext));
  }

  Uint8List _generateNonce() {
    // 使用密码学安全的随机数生成器
    return randomBytes(12);
  }

  // nonce 复用保护
  // 每 2^24 次加密后强制轮换密钥（远超实际用量）
}
```

### 2.6 冲突解决策略

```
冲突场景: 设备 A 和设备 B 同时修改同一对象（离线场景）

检测:
  - 服务端比较 updated_at + content_hash
  - 同版本·不同 hash → 冲突

解决策略（按类型）:

  conversation_history:
    策略: Last-Write-Wins (LWW)
    理由: 同一用户很少同时在两个设备上编辑同一对话
    实现: updated_at 较新者胜出

  mountain_node:
    策略: LWW + 合并提示
    实现: 较新者胜出，旧版本保留为 .conflict 备份

  ombre_bucket (tags/importance):
    策略: CRDT (Conflict-free Replicated Data Type)
    实现: 使用 Observed-Remove Set (OR-Set) 合并 tags
          importance 取平均值

  persona_trait / title / appearance:
    策略: LWW
    理由: 这些数据很少同时修改
```

## 四、匿名聚合引擎

### 4.1 差分隐私方案

```python
class DifferentialPrivacyAggregator:
    """
    使用差分隐私 (ε-DP) 计算全局统计。

    参数:
    - epsilon: 隐私预算 (0.1-1.0)
      越小越隐私，但噪声越大
      推荐: ε=0.5 (平衡隐私与可用性)
    - min_threshold: 最少用户数
      低于此数不报告（防个体推断）
      推荐: 100
    """

    def __init__(self, epsilon: float = 0.5, min_threshold: int = 100):
        self.epsilon = epsilon
        self.min_threshold = min_threshold

    def compute_title_rarity(self, title_id: str) -> TitleRarity:
        """
        计算称号稀有度——不暴露任何个人数据。

        流程:
          1. 统计持有该称号的用户数 (raw_count)
          2. 添加 Laplace 噪声: noisy = raw_count + Laplace(1/ε)
          3. 阈值检查: raw_count < 100 → 不报告
          4. 计算百分比: noisy / total_users
          5. 模糊化输出
        """
        raw_count = self._count_users_with_title(title_id)
        total_users = self._count_total_users()

        # Laplace 噪声
        noise = self._laplace_noise(scale=1.0 / self.epsilon)
        noisy_count = raw_count + noise

        # 阈值检查
        if raw_count < self.min_threshold:
            return TitleRarity(
                title_id=title_id,
                rarity_tier="???",
                message="此称号过于稀有，暂无统计数据",
            )

        # 百分比
        percentage = (noisy_count / total_users) * 100
        percentage = max(0.1, percentage)  # 不低于 0.1%

        return TitleRarity(
            title_id=title_id,
            rarity_percent=round(percentage, 1),
            rarity_tier=self._tier_from_percent(percentage),
            total_users_pool=self._blur_count(total_users),
        )

    def _laplace_noise(self, scale: float) -> float:
        """拉普拉斯分布采样 (位置=0, 尺度=scale)"""
        import secrets
        u = (secrets.randbits(32) / (2**32)) - 0.5
        return -scale * (1 if u < 0 else -1) * math.log(1 - 2 * abs(u))

    def _tier_from_percent(self, pct: float) -> str:
        if pct < 1.0:   return "神话"   # < 1%
        if pct < 5.0:   return "史诗"   # 1-5%
        if pct < 15.0:  return "稀有"   # 5-15%
        return "普通"                    # > 15%

    def _blur_count(self, n: int) -> str:
        """模糊化总用户数"""
        if n < 1000:    return "1000+"
        if n < 5000:    return "5000+"
        if n < 10000:   return "10000+"
        if n < 100000:  return "50000+"
        return "100000+"

    def compute_percentile(self, metric: str, user_value: float) -> PercentileResult:
        """
        计算用户在某指标上的分位数。
        返回: "你的节点数超过了 90% 的同行者"
        """
        raw_percentile = self._calculate_percentile(metric, user_value)
        noise = self._laplace_noise(scale=0.5 / self.epsilon)
        noisy_percentile = raw_percentile + noise
        return PercentileResult(
            metric=metric,
            percentile=round(max(0, min(100, noisy_percentile)), 1),
            message=f"你的{metric}超过了 {noisy_percentile:.0f}% 的同行者",
        )
```

### 7.2 数据生命周期

```
创建 → 活跃 → (请求删除) → 软删除 → 物理删除

活跃期:
  - 用户活跃使用 → 持续同步 → 密文存储 + 聚合贡献
  - 定期备份 (每日增量 + 每周全量)

软删除期 (GDPR: 30天 | PIPL: 15天):
  - 用户请求删除 → deletion_requested_at 标记
  - 账户立即不可用·数据保留·聚合贡献回滚
  - 用户可在此期间恢复账户

物理删除:
  - PostgreSQL: DELETE FROM users/sync_objects WHERE user_id = X
  - 对象存储: 删除所有该用户的 blobs
  - 备份: 下一次备份清理时自然过期
  - 聚合: 重新计算·减去该用户噪声贡献
  - 确认不可恢复 → 写入删除审计日志
```
