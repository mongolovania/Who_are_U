### 1.2 导航设计

采用 **场景切换** 而非传统页面路由——我之山、对话、装扮是三个独立 Scene，之间有转场动画。

```dart
// app_router.dart
final router = GoRouter(
  initialLocation: '/mountain',
  routes: [
    GoRoute(
      path: '/mountain',
      builder: (_, __) => const MountainScreen(),      // 主界面
    ),
    GoRoute(
      path: '/chat',
      pageBuilder: (_, __) => CustomTransitionPage(     // 转场动画
        child: const ConversationScreen(),
        transitionsBuilder: mountainToChatTransition,
      ),

    GoRoute(
      path: '/wardrobe',
      builder: (_, __) => const WardrobeScreen(),

        )

### 4.1 模式切换

```
                    ┌─────────────┐
         App启动      │             │
        ──────────→  │  闲聊模式    │←────────────┐
                     │  (默认)      │              │
                     │             │              │
                     └──────┬──────┘              │
                            │                      │
              [决策按钮] 或   │                      │
           系统检测决策意图   │                      │
                            ▼                      │
                     ┌─────────────┐              │
                     │  决策模式     │──→ 报告生成 ──┘
                     │  5阶段引导    │
                     └─────────────┘
```

### 4.2 决策模式 5 阶段状态机

```
阶段1: 共情倾听
  │  transition: 用户充分表达 + 系统理解就绪
  ▼
阶段2: 自我扫描
  │  transition: 用户完成向内探索
  ▼
阶段3: 选项拆解  ← 注入7位大师视角
  │  transition: 选项分析充分
  ▼
阶段4: 内在确认  ← 大师交叉验证
  │  transition: 用户明确倾向
  ▼
阶段5: 决策输出  ← 大师引用标注
  │
  └──→ 生成决策报告 → 返回闲聊模式
```

## 六、超长上下文管理

### 6.1 策略

```
对话消息列表:
  [m1, m2, m3, ..., m50, m51, m52]

当总 token 数 > maxContextTokens (8000):
  ├── 保留最近 N 条消息（~6K tokens）
  ├── 早期消息 → 压缩为摘要
  └── Memory Palace 已存储的关键节点不受压缩影响
```

### 6.2 压缩触发

  static const int reservedForResponse = 2000;

  }

  Future<String> compress(List<ChatMessage> oldMessages) async {
    // 调用 LLM: "将以下对话压缩为一段摘要，保留关键决策点和情感转折"
    // 返回 ~500 token 的摘要
  }
}
```

### 6.3 记忆浮现（breath）

对话前，Memory Palace 自动浮现相关记忆——但**不刻意说"记得你上次..."**：

```
❌ 生硬: "根据我的记忆，你上次提到过想换工作，现在进展如何？"
✅ 自然: "最近怎么样？" / "今天想聊什么？"
   （记忆信息用于理解上下文、微调回应风格，而非直接引用）
```

仅在对话深入、用户提及相关内容时，才自然地带出——而且是以共情的方式，不是数据库查询的方式。

## 七、加密存储方案

### 7.1 密钥层级

```
用户密码 (v2) / 设备密钥 (v1)
    │
    ▼
主密钥 (SecureStore)
    │
    ├──→ SQLite 加密密钥 (sqlcipher)
    ├──→ FileStore 加密密钥 (AES-256-GCM)
    └──→ v2: E2E 密钥对 (Curve25519·SecureStore)
```

### 7.2 加密文件存储

```dart
class FileStore {
  Future<String> write(String filename, String content) async {
    final key = await _getEncryptionKey();
    final encrypted = AesGcm.encrypt(
      plaintext: content,
      key: key,
      nonce: generateNonce(),  // 12 bytes random
    );
    await File('$path/$filename.enc').writeAsBytes(encrypted);
  }

  Future<String> read(String filename) async {
    final encrypted = await File('$path/$filename.enc').readAsBytes();
    final key = await _getEncryptionKey();
    return AesGcm.decrypt(ciphertext: encrypted, key: key);
  }
}
```
